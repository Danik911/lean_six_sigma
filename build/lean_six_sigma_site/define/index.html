Define Phase

Phase Overview

The Define phase is the first step in the DMAIC methodology. During this phase, we identified and documented the problem, established the project scope, and set clear goals for improvement. The following documents outline the key components of our Define phase.

Key Artifacts

Project Charter

The Project Charter defines the problem statement, project scope, business case, goals, timeline, team members, and stakeholders for the SimplePharma inventory accuracy improvement project. It serves as the foundational document that guides the entire project.

SIPOC Diagram

Download PDF version

The SIPOC (Suppliers, Inputs, Process, Outputs, Customers) diagram provides a high-level overview of the inventory process, identifying key stakeholders and process boundaries.

Swimlane Diagram

Download PDF version

The Swimlane Diagram visualizes the current inventory process flow across different departments and roles, highlighting handoffs and potential bottlenecks.

Define Phase Outcomes

Through the Define phase, we established that inventory discrepancies at SimplePharma are causing significant operational and compliance issues. The project aims to reduce inventory discrepancies by at least 50% within six months, improving operational efficiency and regulatory compliance.

The process mapping revealed several potential problem areas, including manual data entry, lack of standardized procedures, and communication gaps between departments. These insights guided our focus for the measurement phase.
